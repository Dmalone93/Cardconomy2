Three Personas

📦 Buyers & Collectors

Discover, purchase, track, and collect cards. Portfolio management, watchlists, and buylists.

Dashboard: Portfolio value, needs attention, watchlist movements, buylist matches, recent orders

💰 Individual Sellers

List cards, manage offers, ship orders, track earnings. Single and bulk listing flows.

Dashboard: Balance, active listings, pending offers, to ship, activity feed, collection + buylist

🏪 Local Game Shops

Manage walk-in submissions, bulk intake, buylist performance, and daily revenue.

Dashboard: Revenue, submissions pending, bulk lots, buylist hits, submission queue, buylist performance

How to Switch Persona

Mobile: You tab → gear icon → Account type → tap to cycle (buyer → seller → store)
Desktop: Account page → same cycling
Onboarding: First visit shows account type selector
Storage: localStorage cc_acct — persists across sessions

Gaps & Priority Fixes

Issues identified in the user flow audit. Some have already been resolved.

Critical
UX Issue
Missing Feature
Fixed

Critical Gaps

Fixed
No messaging system
Chat/messaging screen added with mock conversation flow.
Critical
No order tracking screen
Purchases screen shows status badges but no dedicated tracking view with timeline/map.
Fixed
Cart not easily accessible
Floating cart pill now appears above tab bar when cart has items.
Critical
No auth/login flow
Everything assumes a logged-in user. No sign up, sign in, forgot password screens.
Critical
No address book
Checkout has one hardcoded address. No saved addresses or address selection.

UX Issues

UX
"Add to my wants" is confusing
Gold button on product page looks like a buy button. Should be "Add to want list" or a secondary action.
UX
Buylist % buttons have no preview
Quick % buttons (80/90/100/110%) don't show what price they calculate to until tapped.
Fixed
Inconsistent quantity selectors
Shared QtyInput, CurrencyInput, and SliderInput components now standardise all inputs.
Fixed
Search had no empty state
Empty state with "clear filters" button now shown when no results match.
UX
No "back to top" on long pages
Product, listing, and search pages can get very long with no quick scroll-back.

Missing Features

Missing
No saved/recent searches
Search shows hardcoded examples but doesn't persist actual user searches.
Missing
No price alerts
Users can watch cards but can't set "notify me when below X" alerts.
Missing
No return/dispute flow
Buyer protection is mentioned but there's no screen to file a claim.
Missing
No shipping label generation
Sellers see "to ship" count but no flow to print labels.
Missing
No multi-currency support
GBP only, no conversion for international buyers.

Priority Ranking

P0 (Ship-blocking): Fix "Add to my wants" label, buylist % preview
P1 (Important UX): Shared input components ✓, Cart badge ✓, Search empty state ✓
P2 (Nice to have): Messaging inbox ✓, Order tracking timeline, Price alerts, Unified seller/store profiles

Demo Tips

• Clear localStorage to reset all state and re-trigger onboarding
• Switch account type via Settings to demo each flow without clearing data
• Scan card works from sell, search, and collection — different primary actions per context
• LiveSweep scanner accessible from sell-to-shop and bulk listing flows
• Trade builder shows real-time fairness % as you select cards
• Price history charts appear on listings with mock historical data
• Offer thread on listings shows mock negotiation with counter-offers
